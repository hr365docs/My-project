General Settings

General settings can be done by only admin role person. By default, the one installed application is an admin, you can assign admin tole to person from the user's option in general settings page.

Custom Settings: From here you can theme color, date format, rename display title for teams and company name, save link validation, Integrate Asset Management Plus with Helpdesk Plus, Graph API Integration and company or brand logo.



  1. Date format: If you change the date format, it will update in entire application.

  2. Display title for teams: If you change the title, it will update in entire application. You can find update display title just below the user's option from where you can create departments or groups. The created one will display in new ticket panel which can be selected during ticket submission.

  3. Company name: If you update company name, it will update in closed and resolved emails notification.

  4. Save link validation: Here you can add the number of the days to the survey link expire.

  5. Tickets can be deleted by: You can allow to delete the ticket other roles person from here.

  6. Integrate Asset Management Plus with Helpdesk Plus: You can integrate HR365 Asset Management application with helpdesk plus application, make sure both applications should be installed in same site location. Once you integrate you will find assigned asset to user in comment page as shown in below. you can click on ticket id in home page to view the comment page.



  7. Graph API integration: If you configure graph API, Ticket mailbox works with help desk email ID.

  8. Use teams code as ticket suffix: Just enable the toggle switch for Use Teams code as ticket suffix

  9. Display resolved tabs in home page: Here you can enable or disable all resolved related tabs in home page.

  10. Brand logo: If you update your company logo, it will update at top of side panel as show in below image.

Users: Admin can assign roles to O365 users with ticket scope as per requirement in Help Desk application. You can select both supervisor and agents for multiple teams but ticket scope will remain same and roles one.



What would happen if user role is agent and he/she is supervisor of team? They will receive notification as a part of supervisor of teams according to Email notification settings and perform activities as per agent roles.

Importance of Roles:


    Admin:
  1. Admin has full control.
  2. Able to do all settings.
  3. Able to raise tickets behalf of another person.
  4. Can generate reports of all teams
  5. Can view dashboard
  6. Able to assign the tickets of all teams
  7. Can view all the teams in side navigation

    Supervisor:
  1. Able to raise tickets behalf of another person.
  2. Receives only raised ticket to their department.
  3. Able to assign the tickets based on ticket scopes
  4. Can generate reports of their teams
  5. Can view dashboard
  6. Can view only those teams in side navigation if they are supervisors of those teams (teams specific)

    Agents:
  1. Able to raise tickets behalf of another person.
  2. They can be able to change the status of their assign tickets and can be able to re-assign to others.
  3. Able to assign the tickets based on ticket scopes

Note: Based on custom settings, admin, supervisor, and agents can delete the tickets

Allow Domains: From here you can allow your orgamization domins users creating tickets from helpdesk email id

Canned Responses

From here admin, supervisor can create responses which will be visible in comment page. There is global canned response, my canned response, and my signature.

Global canned response. This response will display for admin and supervisor in comment page (click on ticket id to open comment page) whoever would be created.
My canned response and my signature: This response will display for admin and supervisor in comments page (click on ticket id from home page to open the comments page) only if they create.
